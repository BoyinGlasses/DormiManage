1. Tổng quan dự án

Repo này là gì, chia layer ra sao, phần nào là runtime và phần nào là tooling.
Core

Sản phẩm chính

Ứng dụng desktop WPF để quản lý ký túc xá cho sinh viên và nhà trường. Đây là bề mặt vận hành chính. Không thấy lớp API ASP.NET trong phần app chính.

WPFMVVMDesktop

Kiến trúc

Repo được chia thành các layer rõ ràng: Domain -> Application -> Infrastructure. Phần WPF dùng các service từ application và wiring qua DI.

DomainApplicationInfrastructureWPF

Phần phụ

dmforum/ là prototype nhỏ bằng Vite/React. Thư mục tests/ chứa test theo từng layer. `graphify-out/` và `.graphify_vendor/` là dữ liệu phục vụ phân tích.

ViteReactxUnit
Cách đọc thực tế: khi mới vào repo, có thể bỏ qua `.graphify_vendor/` và các cache lớn trong `graphify-out/`. Trung tâm của hệ thống nằm ở `src/DormitoryManagement.*` và `tests/`.

7. Luồng xử lý chính

Khởi động, đăng nhập, tải dashboard, điều hướng shell, seed/init.
Flows

Luồng A - Khởi động app

1
App.OnStartup
Dựng host và khởi động app.
2
AppHost.Build
Đọc cấu hình JSON + env, rồi gọi `AddWpfServices`.
3
DbInitializer.InitializeAsync
Đảm bảo có DB và dữ liệu demo.
4
MainWindow.Show
Shell được mở và mặc định đi vào màn hình đăng nhập.

Luồng B - Đăng nhập

1
LoginViewModel.LoginAsync
Validate dữ liệu nhập và xóa lỗi cũ.
2
AuthService.LoginAsync
Tìm user theo email/username/mã sinh viên.
3
Password / lockout / audit
Cập nhật bộ đếm, trạng thái khóa và log audit.
4
Điều hướng theo role
Student -> dashboard, staff -> tickets, role khác -> admin dashboard.

Luồng C - Làm mới dashboard sinh viên

1
StudentDashboardViewModel.LoadAsync
Mở DI scope cho lần tải dữ liệu.
2
Resolve IDashboardService
Lấy service từ service provider trong scope hiện tại.
3
DashboardService.GetStudentDashboardAsync
Ghép DTO từ student, room assignment, room, ticket, invoice.
4
ApplyDashboard
Đưa dữ liệu DTO vào các property để hiển thị.

Luồng D - Điều hướng trong shell

1
NavigateByKey(key)
Switch key sang view-model tương ứng.
2
NavigationService.NavigateTo<T>()
Tạo scope mới và resolve view-model đích.
3
NavigationStore.CurrentViewModel
Giữ view-model đang active.
4
OnNavigationStoreChanged
Shell cập nhật menu/chrome theo route mới.
